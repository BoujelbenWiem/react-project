2. Demarrage rapide

Etape 1 : Installer et lancer l'API

# Dans le dossier api/
npm install                 # Installe json-server localement
npm start                   # Demarre le serveur sur http://localhost:3001
Pas besoin de npm install -g json-server ! Le package.json inclut deja json-server en dependance locale. Un simple npm install + npm start suffit.

Etape 2 : Creer le projet React

# Dans un autre dossier (a cote du dossier api/)
npm create vite@latest mystore-app -- --template react
cd mystore-app
npm install
npm run dev                 # Demarre le projet React sur http://localhost:5173

3. Integration CSS : 3 approches possibles

Pour le style de votre application, vous avez le choix entre 3 approches. A vous de choisir celle qui vous convient le mieux :

Option A
Maquette statique

Reutilisez directement les fichiers CSS fournis dans les maquettes (bootstrap.min.css, style.css, responsive.css) et les classes HTML existantes.

Copiez css/ et fonts/
dans votre dossier
public/ ou src/assets/

Pour : Rapide, rendu identique aux maquettes
Contre : CSS legacy, pas modulaire

Moderne

Option B
SCSS / Sass

Reecrivez le CSS en SCSS avec des variables, des mixins et une architecture par composant. Inspirez-vous des maquettes pour le design.

npm install -D sass
// Creez vos fichiers
src/styles/
├── _variables.scss
├── _mixins.scss
├── components/
└── main.scss

Pour : Maintenable, reutilisable
Contre : Plus de temps

Utility-first

Option C
Tailwind CSS

Utilisez Tailwind CSS pour styler directement dans le JSX avec des classes utilitaires. Inspirez-vous des maquettes pour le design.

npm install -D tailwindcss
  @tailwindcss/vite
npx tailwindcss init

Pour : Rapide, pas de fichiers CSS
Contre : Classes longues dans le JSX

Conseil : Quelle que soit l'approche choisie, ouvrez les maquettes HTML dans votre navigateur pour voir le rendu attendu. Les maquettes sont votre reference visuelle, pas votre source de code.

4. Structure d'un produit

Chaque produit retourne par l'API contient les champs suivants :

{
  "id": 1015,                           // Identifiant unique
  "name": "apple iphone 11 pro max",    // Nom du produit
  "imageName": "apple-iphone-11-pro-max-.jpg",  // Nom du fichier image
  "price": 308,                          // Prix en euros
  "discountRate": 22,                    // Remise en %
  "review": 4,                           // Note de 1 a 5 (etoiles)
  "categoryId": "100",                   // ID de la categorie parente
  "categoryName": "Apple",               // Libelle de la categorie parente
  "type": "phone",                       // phone | tablet | watch
  "description": "Smartphone Apple...",   // Description du produit
  "inStock": true,                       // Disponibilite
  "isTopSeller": false,                  // Afficher dans "Top Sellers"
  "isNewProduct": false                  // Afficher dans "Top New"
}
categoryId + categoryName : Ces deux champs sont presents sur chaque produit pour vous permettre de construire facilement un fil d'Ariane (breadcrumb) et de naviguer entre les categories sans appel API supplementaire.

Calcul du prix apres remise

Prix original :   308 €
Remise (22%) :    308 × 22 / 100 = 67.76 €
Prix final :      308 - 67.76 = 240.24 €

Chemin des images

Les images des produits sont organisees par marque dans produts-img/{Brand}/. Copiez-les dans votre projet React :

// Copiez produts-img/ dans public/images/products/
// Puis referencez-les comme ceci :

/images/products/{categoryName}/{imageName}

// Exemple : /images/products/Apple/apple-iphone-11-pro-max-.jpg

Voici quel endpoint utiliser pour alimenter chaque page :

  • GET /slides → Carousel / Slider
  • GET /products?isTopSeller=true → Section "Top Sellers"
  • GET /products?isNewProduct=true → Section "Top New"

Maquette : shop.html
  • GET /products?categoryId=xxx → Grille produits
  • GET ...&_page=1&_limit=12 → Pagination
  • GET ...&_sort=price&_order=asc → Tri

Page Detail Produit

Maquette : product.html
  • GET /products/:id → Toutes les infos du produit
  • Champs categoryId + categoryName → Fil d'Ariane

Page Panier

Maquette : cart.html
  • GET /carts/:id → Contenu du panier
  • PUT /carts/:id → Modifier quantites
  • DELETE /carts/:id → Vider le panier

Page Checkout

Maquette : checkout.html
  • GET /carts/:id → Recap commande
  • POST /orders → Creer la commande
  • DELETE /carts/:id → Vider apres commande

Barre de Recherche

Presente sur toutes les pages
  • GET /products?q=mot_cle → Recherche globale
  • GET ...?categoryId=xxx&q=mot → Recherche dans une categorie

9. API Cart (Panier) - Cycle de vie complet

Concept cle : json-server n'a aucune notion de session. C'est a vous de gerer l'identifiant du panier. Le principe : faire un POST /carts pour obtenir un id, le stocker (cookie ou localStorage), puis utiliser cet id pour tous les appels GET et PUT jusqu'a la confirmation de la commande.
Exigence technique : La gestion du panier (state, actions, reducers) doit etre implementee avec Redux (Redux Toolkit recommande). Les autres parties du site (categories, produits, recherche, slides) peuvent utiliser des useState / useEffect classiques ou tout autre approche de votre choix.

Cycle de vie du panier

1
POST /carts → Creer le panier Au premier "Add to cart", envoyer un POST pour obtenir un id genere par json-server.
2
Stocker l'id (localStorage ou cookie) Sauvegarder le cartId recu pour l'utiliser dans toutes les requetes suivantes. Le mettre aussi dans un state React.
3
GET /carts/:id → Lire le panier A chaque chargement de page ou refresh, recuperer le panier avec l'id stocke.
4
PUT /carts/:id → Modifier le panier A chaque ajout/suppression de produit ou changement de quantite, envoyer le panier complet mis a jour.
5
POST /orders + DELETE /carts/:id → Confirmer la commande Creer la commande, supprimer le panier cote serveur, puis supprimer le cartId du localStorage et reinitialiser le state. Un nouveau POST sera fait au prochain achat.

Detail des endpoints

POST /carts Creer un nouveau panier

Quand : Premier ajout au panier (quand aucun cartId n'existe dans le storage)

// Envoyez le panier avec le premier produit
// Headers: Content-Type: application/json
// Body :
{
  "total": 240.24,
  "subTotal": 200.20,
  "tax": 40.04,
  "items": [
    {
      "id": 1015,
      "name": "apple iphone 11 pro max",
      "imageName": "apple-iphone-11-pro-max-.jpg",
      "price": 308,
      "qty": 1
    }
  ]
}

// Reponse (json-server genere automatiquement un id) :
{
  "total": 240.24,
  "subTotal": 200.20,
  "tax": 40.04,
  "items": [...],
  "id": "xMrG3rM"         // ← STOCKER CET ID !
}
GET /carts/:id Recuperer le panier

Quand : Au chargement de l'app, si un cartId existe dans le localStorage, recuperer le panier pour l'afficher.

Exemple : GET /carts/xMrG3rM

PUT /carts/:id Mettre a jour le panier

Quand : A chaque modification (ajout produit, +/- quantite, suppression d'un item)

Envoyer le panier complet (pas juste le changement). json-server remplace tout l'objet.

Exemple : PUT /carts/xMrG3rM avec le body complet du panier mis a jour.

DELETE /carts/:id Supprimer le panier

Quand : Apres la confirmation de la commande

Exemple : DELETE /carts/xMrG3rM

Apres cette requete, supprimer le cartId du localStorage et reinitialiser le state du panier a vide.

Rappel : json-server ne fait aucune validation. Si vous envoyez un body vide {}, il creera quand meme un panier avec juste un id. C'est a vous de valider les donnees cote front avant d'envoyer les requetes.

11. Recherche full-text en detail

Comment fonctionne q=

json-server cherche le mot dans tous les champs texte de chaque produit (name, description, categoryName, type). La recherche est insensible a la casse.

ScenarioURLResultats
Recherche globale /products?q=iphone 21 resultats (tous les iPhone, toutes categories)
Recherche dans une categorie /products?categoryId=100&q=ipad 14 resultats (iPad Apple uniquement)
Recherche + type /products?type=tablet&q=samsung Tablettes Samsung
Recherche + pagination /products?q=galaxy&_page=1&_limit=12 12 resultats + header X-Total-Count: 82
Recherche + tri /products?q=xperia&_sort=price&_order=asc 29 resultats tries par prix croissant
Recherche cross-categories /products?q=pro 19 resultats (Apple + Samsung + Huawei)

12. Pagination & Tri

Pagination

json-server supporte la pagination nativement avec _page et _limit.

GET /products?_page=1&_limit=12 Page 1, 12 produits par page

Header de reponse important :

X-Total-Count: 244          // Nombre total de produits (dans le header HTTP)
Link: <...?_page=2&_limit=12>; rel="next",  // Liens de navigation
      <...?_page=21&_limit=12>; rel="last"

Utilisez X-Total-Count pour calculer le nombre total de pages.

Tri

URLDescription
?_sort=price&_order=ascPrix croissant (moins cher d'abord)
?_sort=price&_order=descPrix decroissant (plus cher d'abord)
?_sort=review&_order=descMeilleures notes d'abord
?_sort=name&_order=ascOrdre alphabetique
?_sort=discountRate&_order=descPlus grosses remises d'abord

Combinaison complete

// Produits Samsung, type phone, page 2, 12/page, tries par prix croissant
GET /products?categoryId=c200&type=phone&_page=2&_limit=12&_sort=price&_order=asc

// Recherche "pro" dans toutes les categories, page 1, tries par note
GET /products?q=pro&_page=1&_limit=12&_sort=review&_order=desc
